# 维护文档

## 文件结构

```
服务端（公网机器）:
  server.py          服务端程序
  protocol.py        共享协议模块
  config.json        配置
  server_key.json    RSA 私钥（自动生成）
  server.log         日志

客户端（内网机器）:
  client.py          客户端程序
  protocol.py        共享协议模块
  config.json        配置（自动生成）
  client.log         日志
```

## 日常操作

### 查看日志

```bash
# 服务端
tail -f server.log

# 客户端
tail -f client.log
```

### 修改配置

编辑 `config.json`，重启服务即可。

### 更换服务端密钥

1. 删除 `server_key.json`
2. 重启服务端，自动生成新密钥
3. 复制新公钥给客户端用户
4. 客户端更新 `config.json` 中的 `server_pubkey`

### 更换客户端目标端口

每次启动客户端时输入新的端口即可，无需修改配置文件。

## 故障排查

### 服务端无法启动

- 检查端口是否被占用：`netstat -tlnp | grep 9000`
- 检查 Python 版本：`python --version`（需要 3.7+）

### 客户端连接失败

- 检查服务端是否运行
- 检查防火墙是否开放端口
- 检查 `config.json` 中的 `server_host` 和 `control_port` 是否正确

### 认证失败

- 检查 `config.json` 中的 `server_pubkey` 是否与服务端一致
- 服务端重启后如果重新生成了密钥，需要更新客户端的公钥

### 玩家无法连接

- 检查客户端是否已连接（日志显示 "Tunnel active"）
- 检查本地 MC 服务是否运行
- 检查服务端玩家端口是否开放

## 安全建议

- `server_key.json` 是 RSA 私钥，不要泄露
- 不要将 `server_key.json` 上传到公开仓库
- 定期检查服务端日志，关注异常连接
- 如果怀疑私钥泄露，删除 `server_key.json` 重新生成

## 升级

替换 `server.py`、`client.py`、`protocol.py` 即可，配置文件和密钥文件保持不变。

## 备份

重要文件：
- `server_key.json` — RSA 私钥，丢失后需要重新生成并通知所有客户端
- `config.json` — 配置，丢失后需要重新配置

```bash
# 备份
cp server_key.json server_key.json.bak
cp config.json config.json.bak
```
