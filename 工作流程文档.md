# 工作流程文档

## 整体架构

```
MC 玩家 --> 公网IP:25565 --> [server.py] ===加密隧道=== [client.py] --> 本地MC
```

## 启动流程

### 服务端启动

1. 加载 `config.json`（控制端口、玩家端口）
2. 加载 `server_key.json`（RSA 私钥）
3. 监听控制端口（默认 9000）— 等待客户端连接
4. 监听玩家端口（默认 25565）— 等待 MC 玩家连接

### 客户端启动

1. 加载 `config.json`（服务器地址、公钥）
2. 询问本地 MC 端口
3. 连接服务端控制端口

## 握手流程（RSA 密钥交换）

```
客户端                                  服务端
  |                                       |
  |  <-- nonce(随机数) + RSA签名(nonce) --|  服务端证明身份
  |                                       |
  |  验签（用 config 里的公钥）            |  公钥不对→断开（防中间人）
  |                                       |
  |  -- RSA加密(随机32字节会话密钥) -----> |  只有真服务端能解密
  |                                       |
  |  服务端私钥解密得到会话密钥            |
  |  <-- ok ---------------------------- |  握手完成
  |                                       |
  |  -- 加密(本地MC端口) ---------------> |  告知转发目标
```

- 不需要预共享密码
- 客户端每次生成随机会话密钥
- RSA 加密会话密钥 → 前向保密（每次连接密钥不同）
- 服务端签名 nonce → 客户端验签防中间人

## 数据转发流程

### 帧协议格式

```
[cmd(1字节)] [conn_id(4字节)] [长度(4字节)] [加密数据]
```

| cmd | 含义 |
|-----|------|
| 0x01 | 新连接（通知客户端有新玩家） |
| 0x02 | 数据帧（玩家实际数据） |
| 0x03 | 关闭连接（玩家断开） |

### 单玩家流程

```
MC 玩家                    server.py                    client.py                    本地 MC
    |                          |                            |                            |
    |-- 连接玩家端口 --------->|                            |                            |
    |                          |-- 新连接帧(conn_id=1) --->|                            |
    |                          |                            |-- 连接本地 MC ------------>|
    |                          |                            |                            |
    |-- 玩家数据 ------------>|-- 加密数据帧 ------------>|-- 转发到本地 MC ---------->|
    |<-- 服务端响应 ---------- |<-- 加密数据帧 ------------ |<-- MC 响应 ----------------|
    |                          |                            |                            |
    |-- 玩家断开 ------------>|-- 关闭帧(conn_id=1) ----->|                            |
    |                          |                            |-- 断开本地 MC ------------>|
```

### 多玩家流程

多个玩家通过不同 `conn_id` 复用同一条隧道：

```
玩家1 → conn_id=1 → 隧道 → 客户端 → 本地MC
玩家2 → conn_id=2 → 隧道 → 客户端 → 本地MC
玩家3 → conn_id=3 → 隧道 → 客户端 → 本地MC
```

每个玩家独立的连接，独立的 `conn_id`，通过同一条加密隧道传输。

## 加密机制

### 会话密钥派生

```
随机会话密钥 --[HKDF]--> 服务端发送密钥 + 客户端发送密钥
```

- 服务端 → 客户端的数据用"服务端发送密钥"加密
- 客户端 → 服务端的数据用"客户端发送密钥"加密
- 两个方向的密钥流不同

### 加密算法

SHA-256 CTR 模式：
- 每 32 字节一个密钥块
- 密钥块 = SHA-256(密钥 + 随机数 + 计数器)
- 明文 XOR 密钥块 = 密文

## 断线重连

客户端断线后自动重连：

```
断线 → 等待 1s → 重连失败 → 等待 2s → 重连失败 → 等待 4s → ... → 最长 60s
```

每次重连重新握手，生成新的会话密钥。
