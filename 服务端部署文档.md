# 服务端部署文档

## 文件清单

| 文件 | 说明 |
|------|------|
| `server.py` | 服务端程序 |
| `protocol.py` | 共享协议模块 |
| `config.json` | 配置（首次运行自动生成） |
| `server_key.json` | RSA 私钥（首次运行自动生成） |
| `server.log` | 运行日志 |

## 部署步骤

### 1. 上传文件

将 `server.py` 和 `protocol.py` 上传到公网服务器同一目录。

### 2. 首次运行

```bash
python server.py
```

程序会：
1. 自动生成 RSA 密钥对 → `server_key.json`
2. 自动生成配置文件 → `config.json`
3. 打印服务器公钥

输出示例：
```
  [i] 配置已保存到 /path/to/config.json

  服务器公钥 (复制到客户端 config.json 的 server_pubkey):
  65537:27438571094623847...

  [INFO] Control on 0.0.0.0:9000
  [INFO] Player  on 0.0.0.0:25565
```

**复制公钥**，发给客户端用户。

### 3. 后续运行

```bash
python server.py
```

## config.json

```json
{
  "control_port": 9000,
  "player_port": 25565,
  "control_host": "0.0.0.0",
  "player_host": "0.0.0.0"
}
```

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `control_port` | 9000 | 客户端连接的端口 |
| `player_port` | 25565 | MC 玩家连接的端口 |
| `control_host` | 0.0.0.0 | 控制端口监听地址 |
| `player_host` | 0.0.0.0 | 玩家端口监听地址 |

## 命令行参数

| 参数 | 说明 |
|------|------|
| `--config` | 配置文件路径 |
| `--key-file` | RSA 密钥文件路径 |
| `--control-port` | 控制端口（覆盖配置） |
| `--player-port` | 玩家端口（覆盖配置） |

## 防火墙

```bash
sudo ufw allow 9000/tcp
sudo ufw allow 25565/tcp
```

## 注意事项

- `server_key.json` 是 RSA 私钥，**不要泄露**
- 公钥需要手动复制给客户端用户
- 首次运行后 `config.json` 和 `server_key.json` 不要删除
