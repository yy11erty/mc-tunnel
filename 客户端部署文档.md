# 客户端部署文档

## 文件清单

| 文件 | 说明 |
|------|------|
| `client.py` | 客户端程序 |
| `protocol.py` | 共享协议模块 |
| `config.json` | 配置（首次运行自动生成） |
| `client.log` | 运行日志 |

## 部署步骤

### 1. 准备文件

将 `client.py` 和 `protocol.py` 放到同一目录。

### 2. 首次运行

```bash
python client.py
```

进入配置向导：
```
==================================================
  MC Tunnel - 客户端配置向导
==================================================

公网服务器 IP: 123.45.67.89
端口 [9000]:
服务器公钥 (e:n): 65537:27438571094623847...

  已保存到 config.json
```

配置保存后，每次启动只需输入本地 MC 端口：
```
  [i] 已加载: config.json
本地 MC 端口: 25565
```

### 3. 后续运行

```bash
python client.py
```

## config.json

```json
{
  "server_host": "123.45.67.89",
  "control_port": 9000,
  "server_pubkey": "65537:27438571094623847..."
}
```

| 字段 | 说明 |
|------|------|
| `server_host` | 公网服务器 IP 或域名 |
| `control_port` | 控制端口（和服务端一致） |
| `server_pubkey` | 服务端 RSA 公钥（从服务端复制） |

**不需要密码** — 客户端自动生成随机会话密钥，用服务端公钥加密传输。

## 命令行参数

| 参数 | 说明 |
|------|------|
| `--server-host` | 公网服务器地址 |
| `--target-port` | 本地 MC 端口（跳过询问） |
| `--control-port` | 控制端口 |
| `--config` | 配置文件路径 |
| `--no-reconnect` | 断线不自动重连 |
| `--no-interactive` | 跳过交互式配置 |

## 故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| Connection failed | 服务端没启动或端口不对 | 检查服务端状态 |
| Auth failed | 公钥不对 | 重新从服务端复制公钥 |
| Cannot connect to MC | MC 没启动 | 先启动 MC |
| 频繁断线 | 网络不稳定 | 客户端自动重连（默认开启） |
